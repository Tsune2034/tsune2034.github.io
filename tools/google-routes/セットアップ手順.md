# Google Routes API セットアップ手順

## 概要
旧Distance Matrix API（Legacy化済み）の後継。
複数予約の配達順序最適化に使用。

## 料金
- 無料枠: $200/月クレジット
- Compute Routes: $5/1,000 elements
- Compute Route Matrix: $10/1,000 elements
- **現在の `/api/eta/route.ts` は Directions API を使用中 → Routes APIに移行で精度↑**

## 環境変数（既存）
```
GOOGLE_MAPS_API_KEY=your_key  # 既に設定済み
```

## `/api/eta/route.ts` をRoutes APIに移行する場合

```typescript
// 旧: Directions API（Legacy）
GET https://maps.googleapis.com/maps/api/directions/json

// 新: Routes API（推奨）
POST https://routes.googleapis.com/directions/v2:computeRoutes
Headers:
  X-Goog-Api-Key: {API_KEY}
  X-Goog-FieldMask: routes.duration,routes.distanceMeters,routes.description

Body:
{
  "origin": { "location": { "latLng": { "latitude": 35.7648, "longitude": 140.3861 } } },
  "destination": { "location": { "latLng": { "latitude": 35.6762, "longitude": 139.7649 } } },
  "travelMode": "DRIVE",
  "routingPreference": "TRAFFIC_AWARE_OPTIMAL"
}
```

## Route Matrix（複数予約の最適化）
```
POST https://routes.googleapis.com/distanceMatrix/v2:computeRouteMatrix
→ N origins × M destinations を一括計算
→ 最短経路で複数ホテルを巡回する順序を決定
```
