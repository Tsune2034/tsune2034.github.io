# n8n × KAIROX セットアップ手順

## 概要

n8nをRailwayにセルフホストして、Telegram通知をMac依存から解放する。

```
現状: KAIROX → cloudflared（Macのみ） → Telegram
目標: KAIROX → n8n（Railway・常時稼働）→ Telegram
```

---

## Step 1: RailwayにKAIROX Bridgeをデプロイ

### 1-1. GitHubにpushする

```bash
cd /Users/1hv/ai-company/projects/kairox
git add tools/openclaw-bridge/railway.json
git commit -m "feat: Railway deployment config for telegram bridge"
git push
```

### 1-2. Railwayで新しいサービスを作成

1. https://railway.app → ログイン
2. 「kairox」プロジェクト内 → 「+ New Service」
3. 「GitHub Repo」を選択 → kairoxリポジトリ
4. **Root Directory**: `tools/openclaw-bridge` に変更
5. 「Deploy」

### 1-3. 環境変数を設定（Railway > Service > Variables）

| 変数名 | 値 |
|---|---|
| `TELEGRAM_BOT_TOKEN` | `8753704388:AAG3GR7KMFpweEv_MAIZNyeRBkbtVSbUC70` |
| `OPERATOR_CHAT_ID` | `7002592682` |
| `DRIVER_CHAT_ID` | `6982319714` |
| `OPENCLAW_BRIDGE_SECRET` | `kairox-bridge-secret` |

### 1-4. デプロイ後のURL確認

Railway > Service > Settings > Domains → 自動生成されたURL（例: `xxx.railway.app`）

### 1-5. VercelのOPENCLAW_BRIDGE_URLを更新

```
Vercel > kairox-frontend > Settings > Environment Variables
OPENCLAW_BRIDGE_URL = https://xxx.railway.app
```

## Step 2: n8nをRailwayにデプロイ

### 2-1. Railwayのn8nテンプレートを使う

1. https://railway.app/template/n8n にアクセス
2. 「Deploy Now」をクリック
3. 同じ「kairox」プロジェクトに追加

### 2-2. n8n環境変数を設定

| 変数名 | 値 |
|---|---|
| `N8N_BASIC_AUTH_ACTIVE` | `true` |
| `N8N_BASIC_AUTH_USER` | `kairox` |
| `N8N_BASIC_AUTH_PASSWORD` | （任意の強いパスワード） |
| `N8N_HOST` | `0.0.0.0` |
| `N8N_PORT` | `5678` |
| `WEBHOOK_URL` | `https://n8n.xxx.railway.app/` （デプロイ後のURL） |
| `TELEGRAM_BOT_TOKEN` | `8753704388:AAG3GR7KMFpweEv_MAIZNyeRBkbtVSbUC70` |
| `OPERATOR_CHAT_ID` | `7002592682` |
| `DRIVER_CHAT_ID` | `6982319714` |
| `OPENCLAW_BRIDGE_SECRET` | `kairox-bridge-secret` |
| `KAIROX_API_URL` | `https://kairox.jp` |

## Step 3: n8nワークフローをインポート

### 3-1. n8n管理画面にアクセス

https://n8n.xxx.railway.app → ログイン（Basic Auth）

### 3-2. Telegram認証情報を追加

「Credentials」→「+ New Credential」→「Telegram API」
- Bot Token: `8753704388:AAG3GR7KMFpweEv_MAIZNyeRBkbtVSbUC70`
- 名前: `KAIROX Bot`

### 3-3. ワークフローをインポート

1. 「Workflows」→「Import from File」
2. `tools/n8n/workflows/kairox_telegram_notify.json` をアップロード
3. 「Activate」（緑のトグル）
4. 同様に `kairox_daily_report.json` もインポート・有効化

### 3-4. WebhookのURLを確認してVercelに設定

n8nのTelegram通知ワークフロー → Webhookノードを開く
→ `Production URL`をコピー（例: `https://n8n.xxx.railway.app/webhook/notify`）

Vercelの環境変数を更新:
```
OPENCLAW_BRIDGE_URL = https://n8n.xxx.railway.app/webhook
```
