## 1. GPS地図 全面刷新（Google Maps）

### 問題（v0.12.0 時点）

| 問題 | 症状 |
|---|---|
| マーカー非表示バグ | スタート（S）・現在地マーカーが地図に出ない |
| 目的地なし | 成田空港がどこにあるか地図上で分からない |
| 北上固定 | 走行中に進行方向が分からない |
| 地図が小さい | ミラーリング時に見づらい |

### 解決策

**① レースコンディション修正（コミット: 6fe5e14）**

```
原因: GPS起動時、mapRef.current が null のうちに pointsエフェクトが走り早期return。
     その後マップ初期化が完了してもpoints変化なしで再実行されずマーカー消える。
解法: applyPoints() 共通関数を作成し、init完了直後に pointsRef.current で即描画。
```

**② 成田空港ピン追加**
- 青い「空港」マーカーをGPS起動直後から常時表示
- 全体表示モード（fitBounds）に成田空港を含める

**③ Googleマップナビボタン追加**
- 「🗺 Googleマップでナビ開始（成田空港）」ボタン
- タップ → Google Maps ネイティブアプリが起動・ルート設定済み
- 走行中はアプリ内地図でなくこちらを推奨

**④ ヘディングアップ・ドライブモード（コミット: 680a1cf）**

```typescript
// GPS coords.heading → map.setHeading() で進行方向を常に上に
const newHeading = pos.coords.heading;
setDeliveries(prev => prev.map(d => ({ ...d, heading: newHeading })));

// GpsTrackMap内でheading変化時に回転
useEffect(() => {
  if (!mapRef.current || !headingUp || heading === null) return;
  mapRef.current.setHeading(heading);
}, [heading, headingUp]);
```

| 機能 | 内容 |
|---|---|
| ヘディングアップ | 🧭 北UP / 進行方向UP トグルボタン |
| ドライブモード | 1タップで地図560px + 追従 + ヘディングUP + UI最小化 |
| 角度インジケーター | ドライブモード中に進行方向（度）を右上に表示 |
| マーカー凡例 | "G"ラベル廃止 → 🟢出発地・🟠現在地・🔵成田空港 |

## 2. 双方向プリセットチャット機能

### DBマイグレーション（コミット: f680f35）

```python
# idempotent ALTER TABLE（既存テーブルに安全追加）
migrations = [
    ("bookings", "customer_message", "VARCHAR(32)"),
    ("bookings", "customer_message_at", "TIMESTAMPTZ"),
    ("bookings", "driver_message", "VARCHAR(32)"),
    ("bookings", "driver_message_at", "TIMESTAMPTZ"),
]
```

### バックエンドAPI

```
POST /bookings/{id}/customer-message   ← 顧客→ドライバー
POST /bookings/{id}/driver-message     ← ドライバー→顧客
GET  /bookings/{id}                    ← 全フィールド返却（拡張済み）
```

### 顧客→ドライバー（4種） TrackingView

| キー | メッセージ |
|---|---|
| `coming_out` | もうすぐ出口に出ます |
| `red_bag` | 荷物は赤いスーツケースです |
| `wait_please` | 少し待ってください（5〜10分） |
| `where_driver` | ドライバーはどこですか？ |

- NEWバッジ（受信5分以内でアンバー点滅）
- PATCH `/api/booking?id=` 経由でバックエンドへ中継

### ドライバー→顧客（3種） DriverView

| キー | メッセージ |
|---|---|
| `coming_now` | 今着きました。出口でお待ちです |
| `delayed` | 渋滞中です。あと約10分で到着します |
| `cant_find` | お客様を確認できません。出口でお待ちです |

- 送信後10秒でリセット（再送可能）
- 30秒ポーリングで顧客メッセージを取得・NEWバッジ表示

## 3. 税関ボタン廃止 → 45分カウントダウン（コミット: 24b1c06）

**廃止理由**: 顧客に「ボタンを押す」操作を要求するのは入国直後の混乱時に摩擦が大きい

**代替機能（DriverView）**:
- 監視中フライトの着陸確認 → `landedAt` 記録
- 着陸後45分（平均入国審査+税関所要時間）のカウントダウン
- 0になったら「お客様がそろそろ出口に来る時間です」に変化

## 学習メモ（エンジニア記録）

| 学んだこと | 詳細 |
|---|---|
| Reactレースコンディション | `useEffect([])` と `useEffect([points])` が非同期のため実行順が保証されない。重要な初期描画は init完了後に `ref.current` で直接実行 |
| `coords.heading` の特性 | 静止中は `null` を返す（iOS/Android共通）。走行中のみ 0〜360° で返る |
| `map.setHeading()` | Google Maps JS API でヘディングアップを実現する標準API。引数は北=0° の角度 |
| idempotent ALTER TABLE | 既存テーブルへのカラム追加は try/except + rollback で安全に実行 |
| ドライブモードUX | ミラーリング前提では「1ボタンで全部ONになる」設計が必須 |
