## 2. 実装完了した機能（v0.13.0）

### ① 双方向プリセットチャット

**課題**: 万が一のコミュニケーション手段が必要。自由テキストは走行中に危険なため定型文方式を採用。

**実装**:
```
顧客 → ドライバー（TrackingView）
  [もうすぐ出ます] [赤いスーツケース] [少し待って] [どこにいますか]
  → PATCH /api/booking → POST /bookings/{id}/customer-message

ドライバー → 顧客（DriverView）
  [今着きました] [渋滞中・10分] [お客様が見当たらない]
  → POST /api/driver-message → POST /bookings/{id}/driver-message

30秒ポーリング（双方向）
NEWバッジ（受信5分以内でアンバー点滅）
```

### ② 税関ボタン廃止 → 45分カウントダウン

**議論の経緯**:
- 税関ボタン（顧客が押す）→ 入国直後の混乱時にボタン操作は摩擦が大きい
- 解決策: ドライバーアプリが着陸検知 → 自動45分カウントダウン開始
- 顧客は何もしなくてよい

**残課題（QA指摘）**: 繁忙期（GW・8月）は90〜120分かかる実例あり。45分は「目安」として表示する現実装で当面運用。

### ③ Google Maps 地図改善

| 問題 | 解決 |
|---|---|
| S/Gマーカーが出ない | レースコンディション解消（applyPoints共通関数） |
| 目的地が不明 | 成田空港ピン（青）常時表示 |
| 走行中に方向不明 | ヘディングアップ（進行方向UP） |
| ミラーリング時に小さい | ドライブモード（地図560px・1ボタン） |

### ④ GPS品質

- 15m未満の移動はノイズとして除去
- 最大100点のスライディングウィンドウ
- `coords.heading` 取得 → 進行方向インジケーター
