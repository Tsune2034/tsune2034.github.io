## 1. GPS・ナビ全面強化（DriverView）

### Wake Lock API — 画面スリープ防止

```typescript
// GPS ON時に取得
const wakeLock = await navigator.wakeLock.request("screen");
// GPS OFF時に解放
wakeLock.release();
```

**問題:** スマホのGPS追跡中に画面がスリープ → GPSデータ取得が止まる
**解決:** Wake Lock APIでGPS中は画面を常時点灯

## 2. Telegram通知システム構築

### アーキテクチャ

```
予約確定 → kairox.jp/api/booking
  → cloudflared tunnel
    → localhost:3100 (bridge server)
      → Telegram Bot API
        → 役員・社長のTelegramに届く
```

### bridge server v2.0（OpenClaw廃止）

- `tools/openclaw-bridge/server.js` を全面書き直し
- OpenClaw CLIを排除 → Telegram Bot API直接呼び出し
- `to: "operator"` / `to: "driver"` で送信先を抽象化
- 認証: `X-Bridge-Secret` ヘッダー

### 通知トリガー

| イベント | 送信先 | メッセージ |
|---------|--------|-----------|
| 予約確定 | operator + driver | `📦 [KAIROX] 新規予約 KRX-XXXX / 名前 / プラン / 金額` |
| デイリーレポート（手動） | driver | 当日完了タスクのサマリー |

### Telegramボットメニュー整理

OpenClawの不要コマンド（50+個）を削除 → 3コマンドのみに整理:
- `/report` — デイリーレポート送信
- `/status` — 予約・配送の現状確認
- `/help` — コマンド一覧

## 3. バグ修正・セキュリティ

### NEWバッジ UTCタイムゾーン修正（TrackingView.tsx）

```typescript
// 修正前: ローカル時間として解釈（9時間ズレ）
new Date(info.driverMessageAt).getTime()

// 修正後: Zを付けてUTCとして解釈
const msgTs = ts.endsWith("Z") || ts.includes("+") ? ts : ts + "Z";
new Date(msgTs).getTime()
```

**問題:** バックエンドがUTC（タイムゾーン情報なし）で返す → ブラウザがJST=UTC+9として解釈 → 9時間前のメッセージもNEWバッジが消えない

### GCP APIキー referer制限設定

- HTTPリファラー制限を設定
- 許可URL:
  - `https://kairox.jp/*`
  - `https://*.vercel.app/*`
  - `http://localhost:3000/*`
- 対象API: Maps JavaScript / Directions / Places / Geocoding

## 学習メモ

| 学んだこと | 詳細 |
|-----------|------|
| Wake Lock API | `navigator.wakeLock.request('screen')` でGPS中の画面スリープを防止。`visibilitychange`イベントで再取得が必要 |
| Telegram Bot API削除 | `deleteMessage` APIでbotが送ったメッセージを削除可能。48時間以内のみ有効 |
| UTC naive timestamp | PostgreSQLのTIMESTAMPTZでも、Python側でnaive datetimeを渡すとZなしで返る。フロントで+Zを付けて対応 |
| cloudflared vs ngrok | cloudflaredはアカウント不要・無料でトンネル公開可能。ただしURLはセッション毎に変わる |
