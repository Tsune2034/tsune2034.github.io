# KAIROX Writer Agent — 翻訳・コピーライティングルール

## 役割

このドキュメントは KAIROX の全コピー・翻訳を担当する Writer エージェントのルールブックです。
新しいコピーを追加・変更するとき、必ずこのルールに従ってください。

---

## ブランドボイス

| 要素 | 定義 |
|------|------|
| ブランド名 | **KAIROX**（全言語共通・変更禁止） |
| サブブランド | **Japan Luggage Freedom**（全言語共通・変更禁止） |
| コアコンセプト | 「旅行者に時間を売る」 |
| トーン | 自信・シンプル・スピード感。説明過多にしない |
| 禁止ワード | 「便利」「安心」「最高」「おすすめ」（陳腐・汎用すぎる） |

---

## 言語別ルール

### EN（English）

| ルール | 例 |
|--------|-----|
| 命令形・能動態 | "Land at Narita. Start exploring." ✅ / "You can start exploring" ❌ |
| 短文。1文1アイデア | "No counter, no cut-off time." ✅ |
| 動詞から始める | "Travel Japan hands-free." ✅ |
| カジュアルすぎない | "Your luggage meets you at the hotel." ✅ / "Your bag chills at the hotel" ❌ |
| Oxford comma 使用 | "pickup, delivery, and tracking" |

### JA（日本語）

| ルール | 例 |
|--------|-----|
| 体言止め・余白を活かす | 「着いた瞬間から、旅が始まる。」✅ |
| 丁寧語（です・ます）禁止 | UI上のコピーに限る。チャット返答は除く |
| 読点で間を作る | 「荷物はホテルへ先着。カウンター不要・〆切なし。」✅ |
| カタカナは慎重に | 必要最低限。「ドライバー」「ホテル」はOK |
| 数字は半角 | 「¥1,800」「3人」✅ / 「１，８００円」❌ |

### ZH（中文・简体字）

| ルール | 例 |
|--------|-----|
| 动词先行・简洁 | 「抵达成田，立刻出发。」✅ |
| 繁体字禁止（简体字のみ） | 「行李」✅ /「行李」（繁体）❌ |
| 省略できる主語は省く | 「无需柜台，无截止时间。」✅ |
| 口語・ネットスラング禁止 | 「超方便」❌ |
| 数字は全角禁止 | 「¥1,800」✅ / 「￥１８００」❌ |

### KO（한국어）

| ルール | 例 |
|--------|-----|
| 해요体（〜요）で統一 | 「짐은 호텔에서 만나요。」✅ |
| 합쇼体（〜습니다）禁止 | UIコピーに限る。フォーマルすぎる |
| 名詞句で終わらせてOK | 「카운터 불필요, 마감 없음。」✅ |
| 外来語はカタカナ読みを韓国語音写 | 「드라이버」「호텔」「카운터」 |
| 数字は半角 | 「¥1,800」「3명」✅ |

## 翻訳テンプレート

新しいキーを追加するときは、このフォーマットで4言語セットを作る:

```
キー名:
  EN:
  JA:
  ZH:
  KO:
```

### 例: ヒーローコピー

```
hero:
  EN: Land at Narita. Start exploring. Your luggage meets you at the hotel.
  JA: 着いた瞬間から、旅が始まる。
  ZH: 抵达成田，立刻出发。行李先到酒店。
  KO: 나리타 도착, 바로 여행 시작. 짐은 호텔에서 만나요.

hero_sub:
  EN: No counter, no cut-off time.
  JA: 荷物はホテルへ先着。カウンター不要・〆切なし。
  ZH: 无需柜台，无截止时间。
  KO: 카운터 불필요, 마감 없음.
```

| サブテキスト | EN: 60字 / JA: 30字 / ZH: 24字 / KO: 36字 |

## TRオブジェクトへの追加手順

`app/narita/page.tsx` の `TR` オブジェクトにキーを追加するとき:

1. `en:` ブロックにキー追加
2. `ja:` ブロックに同キー追加
3. `zh:` ブロックに同キー追加
4. `ko:` ブロックに同キー追加
5. `type Tr = { [K in keyof typeof TR.en]: string }` が自動的に型チェック

**必ず4言語同時に追加。1言語でも欠けると TypeScript エラー。**

## NG表現リスト

| NG | 理由 | 代替 |
|----|------|------|
| "Easy" / 「簡単」 | 陳腐 | "In minutes" / 「すぐに」 |
| "Best" / 「最高」 | 根拠なし | 具体的な数値か事実 |
| "Please" / 「ください」 | 指示的・弱い | 命令形・体言止め |
| "We provide" / 「提供します」 | 企業目線 | ユーザー目線に変換 |
| 「等」「など」 | 曖昧 | 具体的に列挙か省略 |
