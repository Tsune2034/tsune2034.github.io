## 会議①：競合調査と価格設計

**テーマ：燃料費ベース価格（¥200/L）・既存競合との差別化**

### 競合調査結果

| サービス | モデル | 課題 |
|---------|--------|------|
| うさぎカーゴ | カウンター受取・翌日配達 | 締め切り14:30〜16:00 |
| Skyporter / Airporter | カウンター型 | 深夜便・遅延便に非対応 |
| JAL ABC / ヤマト | カウンター型・事前予約 | 当日オンデマンド不可 |

**結論：GPS×オンデマンド型は市場に存在しない。本物の空白領域。**

### 価格方針（決定）

- 燃料費（¥200/L）をベースとした実費積み上げ方式
- 高速道路料金は含まない（回避ルート優先）
- ステーブルコイン（JPYC/USDC）利用者に−5%ディスカウント
- クレカ手数料は価格内包（ユーザーへの手数料転嫁なし）← 後の会議で確定

## 会議②：GPS×オンデマンド戦略ピボット

**テーマ：「荷物を持たない北海道旅行」からGPS×オンデマンドモデルへの戦略転換**

### 旧モデルの課題

- 富良野を中心とした北海道観光向け荷物配送
- カウンター受取・事前予約が前提
- 既存競合と本質的な差がない

### 新モデル（GPS×オンデマンド）の定義

**コンセプト：「ドライバーが顧客のいる場所へ来る」**

- 顧客はカウンターに行かない
- GPS または場所指定でドライバーが集荷に来る
- 深夜便・遅延便・突発的なニーズに対応
- 締め切り時間なし・24時間受付

### 展開可能性の評価

| エリア | 評価 | 理由 |
|--------|------|------|
| 成田（NRT） | ◎ 最優先 | 国際線年間3,500万人・外国人比率95%・GPS密度高 |
| 羽田（HND） | ○ 第2フェーズ | 成田で仕組み確立後に横展開 |
| 北海道（CTS） | △ 冬季限定 | 需要分散・季節性リスク・成田知見を転用 |
| 名古屋（NGO） | ○ 将来検討 | 中部圏インバウンド増加中 |

**決定：まず成田で仕組みを検証する。富良野は諦め。**

## 会議③：成田 vs 北海道 — 1人スタートの最適解

**テーマ：1人オペレーター・配送下請け兼業モデルでの最適スタート地点**

### 成田モデルの優位性

**① 検証速度が10倍**
- 成田：1日複数件の実証が可能
- 北海道：需要分散で1日1〜2件が限界

**② 経済的安定性**

| 収入源 | 月次収益 |
|--------|---------|
| AmazonFlex / 佐川 下請け（午前） | ¥120,000 |
| KAIROX（1日2件×20日 @¥5,700平均） | ¥228,000 |
| **合計** | **¥348,000** |
| 車両費・維持費 | −¥40,000 |
| アプリ費用 | −¥5,000 |
| **月次純利益** | **約¥303,000** |

**③ アプリスタックとの適合**
- Aviationstack（フライト情報API）が直接活きる
- マッチング距離が短く（空港内5〜10分圏）GPSロジックが機能する
- 成田→羽田へ同一プロトコルで横展開可能

### 下請け兼業の時間設計

| 時間帯 | 業務 |
|--------|------|
| 08:00〜12:00 | 配送下請け（AmazonFlex / 佐川） |
| 12:00〜18:00 | KAIROX専用（空港周辺待機・受注対応） |
| 夜便対応 | 要判断（深夜便は別途設計） |

**行動半径：成田市〜成田空港〜千葉県北部（車30分圏内）**

### 北海道の扱い

- 捨てるのではなく「シーズン2（冬・スキー期 12〜3月）」として設計
- 成田で3〜6ヶ月間仕組みを磨く → ドライバー1人増やす → 北海道ホテル・スキーリゾートと提携準備 → 12月展開

## 会議④：アプリ設計レビュー

**テーマ：/narita プロトタイプの設計品質評価**

### 評価結果

| 層 | 評価 | 内容 |
|----|------|------|
| 体験層（UI/UX） | 90点 | フロー・デザイン・多言語・スポット選択は競合優位 |
| 信頼設計 | 30点 | 会社情報・補償ポリシー・認証が不足 |
| 実装層（Backend） | 10点 | 予約→通知→確認のループが存在しない |

### 発見された問題点

1. **マッチングがフェイク**：`setTimeout(3500)` のシミュレーション。実際のドライバー通知なし
2. **予約データが保存されない**：フォーム送信後、どこにも記録が残らない
3. **チャットAPIに認証なし**：IPレートリミットなしで無制限にAnthropicコストが発生するリスク
4. **ステップ3（荷物+連絡先+支払い）が過密**：離脱率が高い設計
5. **ドライバー情報がランダム生成**：信頼性の観点から不適切

### 実施した改善

| 優先度 | 対応内容 | ステータス |
|--------|---------|-----------|
| 最優先 | バックエンド接続（POST /bookings → Railway） | 完了 |
| 最優先 | チャットAPIレートリミット（20req/IP/分） | 完了 |
| 最優先 | チャット履歴を最新10件に制限（トークンコスト削減） | 完了 |
| 高 | ステップ分割：luggage（荷物+支払い）→ confirm（連絡先+予約送信） | 完了 |
| 高 | フッターに会社情報・補償ポリシー追加 | 完了 |
| 中 | シングルドライバーモデル（`OPERATOR`固定定数に変更） | 完了 |
| 中 | confirmステップに信頼バッジ（SSL / 補償 / 24hサポート） | 完了 |

### アプリの最終フロー（確定版）

```
pickup（場所選択）
  → destination（届け先選択）
  → luggage（荷物個数 + 支払い方法 + 価格確認）
  → confirm（名前 + 電話 + 注文確認 + Find Driver）
      ↓ POST /bookings → Railway Backend
  → matching（AIルート計算 + ドライバー確認）
  → live（リアルタイム追跡）
```

### 空港スポット設定（追加機能）

Google Mapsでは屋内の場所がわかりにくいという課題に対し、ターミナルごとに具体的なランドマーク（店舗・施設）を選択肢として提供：

| ターミナル | 選択可能スポット |
|-----------|----------------|
| T1（JAL・大韓航空） | 到着ロビー / スターバックス(B1) / セブンイレブン(1F) / バス乗り場 / タクシー乗り場 |
| T2（ANA・ユナイテッド） | 到着ロビー / マクドナルド(1F) / ローソン(B1) / バス乗り場 / タクシー乗り場 |
| T3（LCC各社） | 到着口(1F) / ファミリーマート / フードコート / バス乗り場 |

## 会議⑤：支払い設計の見直し

**テーマ：クレカ+10% / JPYC−5% / USDC−5% の設計は正しいか**

### 問題の本質

- クレカ手数料（実費3.6%）をユーザーに+10%転嫁 → 実費の約2.8倍を請求
- 「後で値段が上がる」体験はUX上の最大アンチパターン
- 訪日外国人の87%がクレカ払いを期待しており、実質的に全員が+10%を払う設計
- Klook/KKday掲載時に「表示価格と違う」レビューが入るリスク

### 競合調査

- JAL ABC、うさぎカーゴ、Grab、Airbnb、Uber — いずれもクレカ手数料をユーザーに転嫁しない

### 決定事項

**「クレカ+10%」を廃止。全支払いの基準価格を統一し「クリプト割引」のみ訴求する。**

#### 変更後の価格設計

| 支払い方法 | 表示 | 実質 |
|-----------|------|------|
| クレカ | 基準価格（追加なし） | 基準価格（カード手数料を内包済み） |
| JPYC | −5%割引バッジ | 基準価格 × 0.95 |
| USDC | −5%割引バッジ | 基準価格 × 0.95 |

#### 価格改定（基準価格 ≒ 旧価格 × 1.04）

| 目的地 | 旧価格 | 新価格 |
|--------|--------|--------|
| 銀座・東京駅 | ¥5,500 | ¥5,700 |
| 新宿・渋谷 | ¥6,000 | ¥6,200 |
| 浅草・上野 | ¥5,000 | ¥5,200 |
| 秋葉原・神田 | ¥5,200 | ¥5,400 |
| 横浜 | ¥7,500 | ¥7,800 |
| さいたま・川越 | ¥7,000 | ¥7,300 |
| 千葉市内 | ¥3,500 | ¥3,700 |
| 羽田空港 | ¥7,500 | ¥7,800 |

#### 財務試算

- 現設計（クレカ+10%）手取り：¥5,832/件（Stripe 3.6%控除後）
- 新設計（手数料内包）手取り：¥5,700 × 0.964 = ¥5,495/件
- **差額：¥337/件** — しかし離脱率が10%減少するだけで月6件増（¥34,200）となり回収可能

## 戦略サマリー

> **「まず成田で仕組みを検証する。1人で始め、下請け兼業で安定収益を確保しながら、GPS×オンデマンドの実証データを積む。冬に北海道へ。」**

- 競合との差別化：カウンター不要・締め切りなし・24時間・ドライバーが来る
- 収益モデル：下請け（午前）+ KAIROX（午後）で月約¥300,000
- 技術的優位：フライト情報×GPS×AI最適ルートの統合
- 拡張経路：成田 → 羽田 → 北海道（冬） → 名古屋
