## このバージョンで何をしたか

### 1. 成田空港 実戦対応機能（P0〜P2）

4/1 成田テスト走行に向けて、アプリのコア機能を強化した。

#### P0: 予約フォームにフライト番号入力フィールド追加（narita/page.tsx）

- confirm ステップにフライト番号入力欄を追加
- 入力すると自動大文字変換（NH847 など）
- 予約POSTボディに `flight_number` を含め、DBの既存カラムに保存
- 注文サマリーにフライト番号を表示

#### P0: DriverView — 着陸アラート + 動的ポーリング（DriverView.tsx）

- FlightBoard に「監視」ボタンを追加。フライト行の右端からタップ可能
- 監視中フライトが ETA ≤ 20分 → アンバー警告「まもなく着陸」
- 着陸済み → 緑「🛬 着陸しました！出発してください」＋実際到着時刻・ターミナル表示
- ポーリング間隔を動的変更：監視中 **2分** / 通常 **10分**（Aviationstack 節約）
- `intervalRef` で複数setIntervalの競合を防止

#### P2: DriverView — 30分無料駐車タイマー（DriverView.tsx）

- 「🚗 入庫 → タイマースタート」ボタンでカウントダウン開始
- 残り 10分 → 赤表示、20分 → アンバー、それ以上 → 緑
- 30分到達で「⚠️ 無料時間終了！出庫してください」に切り替わる
- リセットボタンで再起動可能

#### P2: TrackingView — 「税関を出ました」ボタン（TrackingView.tsx）

- 顧客が予約追跡画面に `CustomsExitButton` コンポーネントを追加
- 4言語対応：EN / JA / ZH / KO
- タップ → `PUT /api/booking` に `{ customs_exited: true }` を送信
- 「✅ ドライバーに通知しました」に切り替わる

## 学習メモ（エンジニア記録）

| 学んだこと | 詳細 |
|---|---|
| Vercel env var のスペース問題 | GUI入力時に末尾スペースが混入することがある → 全PIN/SECRET比較は `.trim()` 必須 |
| Aviationstack free プランの節約設計 | 500req/月 → 監視中フライトのみ2分、それ以外10分が最適バランス |
| intervalRef パターン | React の setInterval は cleanup しないと複数起動する → `useRef` で管理 |
| 成田空港無料駐車 | P1〜P5 全て30分無料（2025/4/1〜）。入庫タイマーで ¥0 運用が可能 |
| GPS学習の完全フロー | `GpsTrackPoint`（30秒/点）→ 配送完了で `_analyze_route_on_complete()` → `RouteStats`（補正係数）→ 次回ETA改善 |
| Pydantic extra fields | FastAPI は body の未定義フィールドをデフォルトで無視。`customs_exited` は次回マイグレーション時に追加予定 |
