## このバージョンで何をしたか

### 概要

「Telegram で運行管理する」フェーズ。ドライバーとオペレーターが Telegram Bot から全ての運行業務（点呼・GPS・予約管理・メーター記録）をこなせる仕組みを構築。同時に BookingApp をゾーン共通コンポーネント化し、千歳空港に対応。インメモリ状態を完全廃止し Railway PostgreSQL で永続化。

## 学習メモ

| 学んだこと | 詳細 |
|---|---|
| Vercel Serverless とインメモリ | リクエストごとに別インスタンスの可能性。モジュールレベル変数は信頼できない → 必ず外部DBへ |
| Telegram Webhook vs Polling | setWebhook と getUpdates は排他。OpenClaw が getUpdates していると Webhook が無効化される |
| launchd の自動再起動 | `launchctl stop` だけでは再起動される。`unload` で plist ごと無効化が必要 |
| Telegram コマンドメニュー | `setMyCommands` で `/` ボタン押下時のメニューを設定。OpenClaw が上書きしていたので再設定が必要 |
| /cmd@botname 形式 | メニューから送ると `/duty_on@kairox_notify_bot` になる。`split("@")[0]` で除去が必要 |
| next-pwa + Turbopack | next-pwa は内部で webpack 設定を注入。Next.js 16 Turbopack と競合 → `turbopack: {}` で解消 |
